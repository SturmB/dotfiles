# Note Maintenance

## Overview

Ensure every response that produces, gathers, or discusses substantive information results in vault notes being created or updated. This skill exists because passive instructions in CLAUDE.md get forgotten in the flow of answering questions.

**Core principle:** Check before you finish. Every time.

## When to Use

- Every conversation in the Obsidian vault
- Applies to ALL responses, not just task completion

**When a response produces NO notable information** (pure Q&A, clarifying questions, conversation about approach), no notes are needed — but you must still consciously check.

## The Gate

Before finishing any response, ask yourself:

> "Did this response produce, gather, or discuss information that belongs in the vault?"

**If yes:** Create or update notes following CLAUDE.md conventions before finishing.
**If no:** Move on.

## What Counts as Notable

- Configuration changes, fixes, or troubleshooting steps
- Research results or decisions made
- New tools, services, or setup details
- Status changes (migrations, removals, upgrades)
- Anything you'd want to find again in a future session

## What Doesn't Count

- Purely conversational back-and-forth
- Clarifying questions
- Interim brainstorming before decisions are made
- Information already captured in the vault

## Common Mistakes

| Mistake | Fix |
|---------|-----|
| Getting absorbed in answering and forgetting notes | The gate is AFTER answering — answer first, then check |
| Creating a new note when one already exists | Always search for existing notes on the topic first |
| Dumping notes in Inbox | Route to the correct area directory per CLAUDE.md |
| Skipping because "it's minor" | Minor config changes are the most useful to find later |
| Writing notes but not adding wiki-links | Connect new content to related existing notes |

## Red Flags

These thoughts mean you're about to skip note maintenance:

- "I'll note this later" — No. Note it now.
- "This isn't important enough" — Config changes, fixes, and decisions always are.
- "The user didn't ask me to" — CLAUDE.md says to do it proactively.
- "I already told the user the answer" — Telling is not noting. Vault notes persist; chat doesn't.
